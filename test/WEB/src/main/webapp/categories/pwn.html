주요 문제 유형 & 공략법

  • Buffer Overflow

    정의: 입력값이 버퍼 크기를 초과해 인접 메모리(리턴 주소 등)를 덮어쓰는 취약점
    공격 가능한 조건:
    • 입력값 길이 제한이 없는 gets(), scanf(), strcpy() 등 사용
    • 스택에 저장된 리턴 주소 등 중요한 데이터가 보호되지 않을 때
    공격 방법:
    • 과도한 입력값으로 리턴 주소를 원하는 값(쉘코드, 함수 주소 등)으로 덮어쓰기
    • Exploit: python -c 'print("A"*offset + "B"*4)' | ./vuln

  • Format String

    정의: printf 등 포맷 함수에 사용자의 입력이 직접 들어가 메모리 읽기/쓰기 등이 가능한 취약점
    공격 가능한 조건:
    • printf(user_input)처럼 입력값이 포맷 문자열로 사용될 때
    공격 방법:
    • %x, %s, %n 등 포맷 스트링을 이용해 메모리 값 유출/변조
    • 예: AAAA%x%x%x%x 입력 시 스택 값 노출

  • Use-After-Free

    정의: 해제된 메모리(포인터)를 다시 사용해 임의 코드 실행 또는 정보 유출이 가능한 취약점
    공격 가능한 조건:
    • free() 호출 후 포인터를 재사용
    • 메모리 재할당 시 공격자가 원하는 데이터로 덮어쓰기 가능
    공격 방법:
    • 해제 후 재할당된 영역에 악의적 데이터 삽입
    • 함수 포인터, vtable 등 중요한 포인터 덮어쓰기

  • ROP (Return Oriented Programming)

    정의: 코드 실행 방지 기법(DEP, NX)을 우회하기 위해 기존 바이너리 내 명령어 조각(gadget)을 연결해 공격하는 기술
    공격 가능한 조건:
    • 스택 오버플로우 등으로 리턴 주소 제어 가능
    • 실행 가능한 코드 조각(gadget)이 존재
    공격 방법:
    • gadget 주소들을 스택에 쌓아 원하는 시스템 호출 등 수행
    • 예: system("/bin/sh") 실행을 위한 ROP 체인 생성

주요 도구 & 툴

  • pwntools

    설명: 파이썬 기반의 바이너리 익스플로잇 자동화 라이브러리
    주요 기능:
    • 원격/로컬 바이너리와 통신, 페이로드 자동 생성
    • ROP 체인, 쉘코드, 패딩 자동화
    활용 예시:
    • 익스플로잇 스크립트 작성, 반복 공격 자동화

  • GDB (GNU Debugger)

    설명: 리눅스 환경에서 바이너리 디버깅, 메모리/레지스터/스택 분석 도구
    주요 기능:
    • 브레이크포인트, 메모리/레지스터 값 확인 및 수정
    • 프로그램 흐름 추적, 스택 프레임 분석
    활용 예시:
    • 오버플로우 위치, 페이로드 효과, ROP 체인 동작 확인

  • IDA Pro / Ghidra / Radare2

    설명: 바이너리의 어셈블리 코드, 함수 흐름, 제어 흐름 그래프를 분석하는 리버싱 툴
    주요 기능:
    • 디스어셈블, 함수/변수 분석, 코드 주석
    • 자동화 스크립트, 플러그인 지원
    활용 예시:
    • 익스플로잇 대상 함수/버그 위치 파악, 바이너리 구조 분석

  • one_gadget

    설명: libc 파일에서 system("/bin/sh") 실행이 가능한 gadget 주소를 자동으로 찾아주는 도구
    주요 기능:
    • libc 버전별 one-gadget 주소 자동 검색
    활용 예시:
    • libc leak 후 익스플로잇에 바로 활용
