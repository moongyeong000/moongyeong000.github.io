주요 문제 유형 & 공략법

  • 패킷 분석

    정의: 네트워크 트래픽(pcap 등)에서 플래그, 인증 정보, 평문 데이터 등을 찾는 문제
    공격 가능한 조건:
    • 캡처된 네트워크 트래픽 파일 제공
    공격 방법:
    • Wireshark 등으로 프로토콜/스트림 분석, 파일/플래그 추출

  • 서비스 분석

    정의: 서버에서 동작 중인 서비스의 취약점/정보를 분석하는 문제
    공격 가능한 조건:
    • 포트가 열려 있고, 서비스 배너/응답이 노출되는 경우
    공격 방법:
    • nmap 등으로 서비스/버전 식별, 취약점 검색
    • 취약 서비스에 직접 연결하여 플래그 획득

  • Port Knocking / Custom Protocol

    정의: 특정 순서로 포트에 접속하거나, 커스텀 프로토콜을 분석하는 문제
    공격 가능한 조건:
    • 문제에서 포트 시퀀스, 커스텀 프로토콜 힌트 제공
    공격 방법:
    • netcat, nmap, 스크립트 등으로 지정된 포트/프로토콜에 맞춰 접속

주요 도구 & 툴

  • Wireshark

    설명: 네트워크 패킷 분석 및 프로토콜 디코딩 도구
    주요 기능:
    • TCP/UDP/HTTP 등 다양한 프로토콜 분석
    • 필터링, 스트림 재조립, 파일 추출
    활용 예시:
    • 패킷 내 플래그, 파일, 인증 정보 추출

  • nmap

    설명: 네트워크 포트 스캐닝 및 서비스/버전 탐지 도구
    주요 기능:
    • 열려 있는 포트, 서비스, 운영체제 탐지
    • 스크립트 엔진으로 취약점 자동 진단
    활용 예시:
    • 서비스 정보 수집, 취약점 탐지, 포트 노킹 테스트

  • netcat (nc)

    설명: TCP/UDP 연결 및 데이터 송수신, 포트 스캐닝 등 다용도 네트워크 도구
    주요 기능:
    • 서버/클라이언트 연결, 데이터 전송, 포트 리스닝
    활용 예시:
    • 원격 서버 접속, 커스텀 프로토콜 테스트, 플래그 획득

  • tcpdump

    설명: 터미널 기반 네트워크 패킷 캡처 및 분석 도구
    주요 기능:
    • 실시간 트래픽 캡처, 필터링, 패킷 저장(pcap)
    활용 예시:
    • 패킷 캡처 후 Wireshark 등으로 상세 분석
