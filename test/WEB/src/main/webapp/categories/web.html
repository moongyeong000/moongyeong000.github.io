주요 문제 유형 & 공략법

  • SQL Injection

    정의: 웹 애플리케이션의 입력값이 SQL 쿼리에 직접 삽입되어, 공격자가 쿼리를 조작할 수 있는 취약점
    공격 가능한 조건:
    • 입력값 검증/필터링이 미흡한 로그인, 검색, 게시판 등
    • 입력값이 DB 쿼리에 직접 사용되는 경우
    공격 방법:
    • 입력창에 ' OR 1=1 -- 등 SQL 구문 삽입
    • Blind SQLi: 참/거짓에 따라 페이지 반응을 분석
    • Union-based: UNION SELECT로 데이터 추출

  • XSS (Cross-Site Scripting)

    정의: 악성 스크립트가 웹 페이지에 삽입되어 사용자의 세션/정보를 탈취할 수 있는 취약점
    공격 가능한 조건:
    • 사용자 입력이 필터링 없이 HTML에 반영되는 경우
    • 댓글, 게시판, URL 파라미터 등 동적 콘텐츠
    공격 방법:
    • 입력값에 <script>alert(1)</script> 등 삽입
    • 쿠키 탈취, 세션 탈취, 피싱 페이지 유도 등

  • File Upload

    정의: 파일 업로드 기능에서 필터링 미흡으로 웹쉘 등 악성 파일이 업로드되는 취약점
    공격 가능한 조건:
    • 확장자/콘텐츠 타입 검증이 불충분한 업로드 기능
    • 업로드 파일이 웹에서 실행/접근 가능할 때
    공격 방법:
    • 우회 기법(.php.jpg, .ph%00p 등)으로 웹쉘 업로드
    • 업로드된 파일 경로로 직접 접근하여 명령 실행

  • Directory Traversal

    정의: 파일 경로 조작을 통해 서버 내 임의 파일에 접근하는 취약점
    공격 가능한 조건:
    • 파일명에 ../ 등 상대경로가 필터링되지 않는 경우
    • 다운로드/이미지 뷰어 등에서 파일명을 입력받을 때
    공격 방법:
    • 입력값에 ../../../../etc/passwd 등 삽입
    • 서버 내 중요한 파일 노출/다운로드

  • Authentication Bypass

    정의: 인증 로직의 허점을 이용해 비정상적으로 로그인을 우회하는 취약점
    공격 가능한 조건:
    • 약한 비밀번호, 디폴트 계정, 인증 우회 로직
    • 세션/쿠키 조작, 파라미터 변조
    공격 방법:
    • SQLi, 논리 오류, 세션 고정 등 다양한 방식으로 인증 우회
    • 관리자 페이지 접근, 권한 상승

  • SSRF (Server-Side Request Forgery)

    정의: 서버가 공격자가 지정한 URL로 요청을 보내도록 유도하는 취약점
    공격 가능한 조건:
    • 이미지/URL 다운로드 등 서버가 외부 요청을 수행하는 기능
    • 내부망/로컬 주소 필터링 미흡
    공격 방법:
    • 입력값에 http://localhost/admin 등 내부 주소 지정
    • 클라우드 메타데이터, 내부 서비스 접근

주요 도구 & 툴

  • Burp Suite

    설명: 웹 프록시로 모든 HTTP/S 트래픽을 가로채고, 요청/응답을 자유롭게 수정할 수 있는 해킹/침투 테스트 도구
    주요 기능:
    • Proxy: 실시간 요청/응답 가로채기 및 수정
    • Repeater: 요청 반복/변경 테스트
    • Intruder: 자동화된 파라미터 공격, 브루트포싱
    • Decoder: 인코딩/디코딩, 해시 변환
    • Scanner: 취약점 자동 탐지(유료)
    활용 예시:
    • 로그인 우회, SQLi/XSS 페이로드 테스트, 파일 업로드 조작 등 다양한 웹 해킹 실습에 필수

  • OWASP ZAP

    설명: 오픈소스 웹 취약점 진단 도구로, Burp Suite와 유사한 기능 제공
    주요 기능:
    • 자동/수동 스캔, 프록시, 파라미터 조작, 페이로드 자동화
    • 다양한 플러그인과 커뮤니티 지원
    활용 예시:
    • 무료로 웹 취약점 진단을 체험하고 싶은 초보자에게 적합

  • SQLMap

    설명: SQL Injection 자동 탐지 및 익스플로잇 도구
    주요 기능:
    • 다양한 DBMS 지원, 데이터베이스 구조 자동 추출
    • DB 계정, 테이블, 컬럼, 데이터 자동 덤프
    • 블라인드/에러 기반/Union 기반 등 다양한 공격 지원
    활용 예시:
    • SQLi가 의심되는 URL을 입력하면 자동으로 공격 및 데이터 추출

  • Postman

    설명: REST API 테스트 및 자동화 도구
    주요 기능:
    • HTTP 요청 생성/전송, 파라미터/헤더/바디 자유 조작
    • API 시나리오 자동화, 응답 분석
    활용 예시:
    • API 인증 우회, SSRF, 파라미터 변조 등 테스트에 활용

  • Dirbuster / Dirsearch

    설명: 웹 서버의 숨겨진 디렉토리/파일을 브루트포싱으로 찾는 도구
    주요 기능:
    • 워드리스트 기반 디렉토리/파일 탐색
    • 응답 코드/길이 분석으로 존재 여부 판단
    활용 예시:
    • admin, .git, 백업파일 등 숨겨진 경로 찾기

  • CyberChef

    설명: 다양한 인코딩, 디코딩, 암호화, 변환을 웹에서 쉽게 할 수 있는 도구
    주요 기능:
    • Base64, URL, Hex, ROT13 등 다양한 변환 지원
    • 여러 변환을 조합하여 한 번에 처리 가능
    활용 예시:
    • 문제에서 받은 인코딩/암호화 데이터를 빠르게 복호화
