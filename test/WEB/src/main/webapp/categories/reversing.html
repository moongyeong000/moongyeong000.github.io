주요 문제 유형 & 공략법

  • CrackMe

    정의: 특정 입력(시리얼, 패스워드 등)을 맞추는 바이너리 분석 문제
    공격 가능한 조건:
    • 바이너리 내에 정답 검증 로직이 존재
    공격 방법:
    • 디스어셈블/디버깅으로 비교 지점, 정답 데이터 추출
    • 패치, 브루트포싱, 입력 변환 등 활용

  • 암호화/디코딩 분석

    정의: 바이너리 내에 숨겨진 인코딩/암호화 알고리즘을 분석해 원본 데이터를 복원하는 문제
    공격 가능한 조건:
    • 코드 내에 인코딩/암호화 함수가 존재
    공격 방법:
    • 알고리즘 분석, 복호화 스크립트 작성

  • Anti-Debug/Anti-Reversing

    정의: 리버싱/디버깅을 방해하는 기술이 적용된 바이너리 분석 문제
    공격 가능한 조건:
    • 디버거 감지, 난독화, 코드 흐름 변경 등
    공격 방법:
    • 패치, 우회, 스크립트/플러그인 활용

주요 도구 & 툴

  • IDA Pro / Ghidra

    설명: 바이너리의 어셈블리/함수 흐름을 시각적으로 분석하는 리버싱 툴
    주요 기능:
    • 디스어셈블, 함수/변수 분석, 제어 흐름 그래프
    • 자동화 스크립트, 플러그인 지원
    활용 예시:
    • 정답 검증/암호화 함수 분석, 패치 포인트 찾기

  • x64dbg / OllyDbg

    설명: 윈도우 환경에서 동적 디버깅을 지원하는 도구
    주요 기능:
    • 브레이크포인트, 메모리/레지스터 값 확인, 실시간 코드 실행 추적
    활용 예시:
    • 입력값 검증 우회, 동적 분석, 패치 테스트

  • angr

    설명: 파이썬 기반의 바이너리 분석 및 심볼릭 실행 프레임워크
    주요 기능:
    • 경로 탐색, 자동화된 입력값 생성, 취약점 분석
    활용 예시:
    • 복잡한 CrackMe, 조건 분기 자동화 분석
