소개

안녕하세요. 한남대학교 컴퓨터공학과 2학년, 공격형 해킹 기업에서 레드팀/취약점 분석 전문가를 꿈꾸는 정문경입니다.

코딩을 시작하게 된 계기


제 첫 코딩은 중학교 2학년 당시 수학 선생님의 권유로 하루에 30분씩 알고리즘을 배우던 것에서 시작했습니다. 정보 올림피아드에 나갈 사람을 구하고 계셨었고 운 좋게 눈에 띄어 배웠지만 그 당시에는 별 생각이 없었습니다. 고등학교 진학 후 정보 과목에서 제대로 배워보며, 이 길이 내 길이구나를 직감했습니다. 다른 아이들에 비해 잘하는 유일한 과목이었기 때문입니다. 하지만 혼자의 힘으로는 정보 수집부터 공부하는 방법과 무엇을 공부해야하는지, 어디서 정보나 연습이 가능한지 모든 걸 몰랐기에 대학 진학 후 진짜 코딩을 해봤다고 생각합니다. (코드업 100제를 풀어봤으나 기초 수준들이기에 큰 성장은 느끼지 못했습니다.)

보안 전문가를 선택한 이유


예전부터 남을 도우며 사는 것이 꿈이었습니다. 그렇기에 초등학교 떄는 변호사, 중학교 떄는 군인이었다가 고등학교 2학년 끝 무렵에 보안 전문가라는 길을 선택했습니다. 코딩을 조금 잘했지만 자신이 없어 안정적인 군인을 하려 했으나 코딩이 생각보다 재미있고, 즐기면서 일 할 수 있겠다는 생각이 들어 진로를 변경했습니다. 개발자가 아닌 보안 전문가인 이유는 나를 희생해 남을 돕는다는 제 희망에 더 가까웠고, 당시에도 인터넷상에서의 불법적인 일들이 너무나 많이 일어나고 있었기에 이를 막고자 선택하게 되었습니다.

관리자 페이지 (SQL Injection)

SQL Injection을 통해 로그인에 성공하였습니다.
SQL Injection이란?
사용자 입력값이 SQL 쿼리에 그대로 삽입되어 인증 우회, 데이터 탈취, 시스템 명령 실행 등 다양한 공격이 가능합니다.
예시 페이로드: ' OR 1=1 --
대응 방안: 입력값 검증, Prepared Statement 사용 등

CSRF(사이트 간 요청 위조)란?

CSRF는 인증된 사용자의 권한을 도용해 악의적인 요청을 보내는 공격입니다.
대응 방안: CSRF 토큰 사용, Referer/Origin 검증, SameSite 쿠키 설정 등

매니저 페이지 (URL 파라미터 우회)

URL 파라미터 우회로 인증을 우회하였습니다.
개념: URL에 ?manager=true 등 특정 파라미터를 추가해 인증/인가를 우회하는 공격
대응 방안: 서버단 인증/인가 로직 필수, 관리자 페이지 별도 인증/접근제어
