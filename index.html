로그인 실습

SQL Injection 또는 URL 파라미터 조작(예: ?manager=true)으로 인증 우회 실습이 가능합니다.

관리자 페이지 (SQL Injection)

SQL Injection을 통해 로그인에 성공하였습니다.
SQL Injection이란?
사용자 입력값이 SQL 쿼리에 그대로 삽입되어 인증 우회, 데이터 탈취, 시스템 명령 실행 등 다양한 공격이 가능합니다.
예시 페이로드: ' OR 1=1 --
대응 방안: 입력값 검증, Prepared Statement 사용 등

CSRF(사이트 간 요청 위조)란?

CSRF는 인증된 사용자의 권한을 도용해 악의적인 요청을 보내는 공격입니다.
대응 방안: CSRF 토큰 사용, Referer/Origin 검증, SameSite 쿠키 설정 등

매니저 페이지 (URL 파라미터 우회)

URL 파라미터 우회로 인증을 우회하였습니다.
개념: URL에 ?manager=true 등 특정 파라미터를 추가해 인증/인가를 우회하는 공격
대응 방안: 서버단 인증/인가 로직 필수, 관리자 페이지 별도 인증/접근제어

경고: 실제 서비스에서는 이런 취약점이 존재하지 않도록 반드시 서버단에서 권한을 체크해야 합니다.
