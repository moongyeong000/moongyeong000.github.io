관리자 페이지 (SQL Injection)

SQL Injection을 통해 로그인에 성공하였습니다.
SQL Injection이란?
사용자 입력값이 SQL 쿼리에 그대로 삽입될 때, 논리 연산자(OR 1=1)나 주석(--)을 활용해 인증 우회, 데이터 탈취, 시스템 명령 실행 등 다양한 공격이 가능합니다.
예시 페이로드: ' OR 1=1 --
대응 방안: 입력값 검증, Prepared Statement 사용 등

참고 : CSRF(사이트 간 요청 위조)란?

CSRF는 인증된 사용자의 권한을 도용해 악의적인 요청을 보내는 공격입니다.
예를 들어, 사용자가 은행 사이트에 로그인된 상태에서 공격자가 만든 악성 웹페이지를 방문하면, 해당 페이지에서 사용자의 계정으로 이체 요청 등 중요한 동작이 실행될 수 있습니다.
대응 방안: CSRF 토큰 사용, Referer/Origin 검증, SameSite 쿠키 설정 등

매니저 페이지 (URL 파라미터 우회)

URL 파라미터 우회로 인증을 우회하였습니다.
개념:서버가 클라이언트가 전달하는 파라미터만으로 인가를 판단할 때, URL에 ?manager=true 등 특정 파라미터를 추가하는 등의 값을 조작해 권한이 없는 리소스에 접근할 수 있는 공격
대응 방안: 서버단 인증/인가 로직 필수, 관리자 페이지 별도 인증/접근제어
